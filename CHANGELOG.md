# Changelog

## [1.5.0] - 2026-03-11

### Added
- **Вкладка «Товары»** — новый модуль жалоб на товары со страницы `brand-products/all`
- **`content-products.js`** — отдельный content script для парсинга таблицы товаров (7 колонок: название+SKU, дата, бренд, продавец, цена покуп., цена продавца, меню)
- **Настройки жалобы на товар** — отдельный текст жалобы (`productComplaintText`) и файл (`productFileData`) для товарных жалоб
- **Фильтр по Whitelist** — товары продавцов из Whitelist пропускаются (зелёная подсветка), остальные — жалоба (красная)
- **Авто-навигация** — кнопки «Сканировать» и «Найти бренды» автоматически открывают нужную страницу OZON
- **`ensureOzonPage()`** — универсальная навигация: ищет вкладку → навигирует текущую → создаёт новую
- **`ensureProductsPage()`** — навигация на `brand-products/all`
- **`ensureProductContentScript()`** — инжект `content-products.js`
- **project.md** — полная карта проекта: архитектура, функции, селекторы, config schema

### Changed
- **manifest.json** — добавлен content_scripts match для `brand-products/*`
- **service-worker.js** — relay: `updateProductStats`, `doneProducts`
- **popup.html** — 7 вкладок (+ Товары), `btn--warning` стиль
- **popup.js** — `DEFAULT_CONFIG` расширен: `productComplaintText`, `productFileData`, `productFileName`
- **Плавающая панель товаров** — оранжевая тема (отличается от синей sellers)

## [1.4.0] - 2026-02-14

### Added
- **Вкладка «Жалобы» с таблицей** — полная таблица: дата, продавец, ИНН, бренд, страна, статус (OK/Ошибка)
- **Экспорт жалоб** — 3 формата: CSV (с BOM для Excel), Excel (.xls), TXT (табличный)
- **Лимит истории 5000 записей** — счётчик «X / 5000» в заголовке
- **Под-вкладка «Техническая»** — все технические логи (DIAG, клики, поиск) в отдельной вкладке с тёмной консолью
- **Красная подсветка бренда без документа** — если у бренда нет прикреплённого файла, карточка подсвечивается красным
- **Синий щит** — новая иконка SVG (щит с галочкой) в header и `assets/icon.svg`
- **`techLog` relay** — content.js отправляет каждую строку лога в popup через background

### Changed
- **Автопоиск брендов** — убраны лишние описания, оставлена только кнопка «Найти бренды на странице»
- **Вкладка «Лог» → «Жалобы»** — переименована, показывает только результаты жалоб (не технические логи)
- **Вкладка «Инструкция» → «Инфо»** — сокращена, обновлены рекомендации по задержке (15-30с)
- **Описания упрощены** — убраны длинные hint-тексты из Whitelist, Brands, Guide

## [1.3.3] - 2026-02-14

### Fixed
- **Sidebar не открывался — PointerEvents** — `simulateRealClick()` теперь отправляет полный цикл: pointerover→pointerenter→pointerdown→pointerup + mouseover→mouseenter→mousedown→mouseup→click + native `.click()`. React 17+ использует PointerEvents для event delegation
- **Кнопка СТОП не останавливала полностью** — добавлены проверки `shouldStop` в каждом async шаге: `fileComplaint()`, `findTextareaInModal()`, `findComplainButtonWithRetry()`, `waitCooldown()`. Стоп теперь закрывает модалы и отправляет Escape
- **Отладка клика «Пожаловаться»** — в лог выводится полная информация о кликаемом элементе: `<tag role class> "text"` и `Parent: <tag class>` — это покажет правильный ли элемент кликается
- **Submit тоже через simulateRealClick** — кнопка «Отправить» теперь кликается через полный цикл событий вместо `.click()`

## [1.3.2] - 2026-02-14

## [1.3.0] - 2026-02-14

### Changed
- **Все парсеры переписаны по реальному HTML OZON** — пользователь предоставил 3 фрагмента DOM (строка таблицы, модал жалобы, пагинация), все функции переписаны с точными селекторами
- **`parseSellersTable()`** — использует `table tbody tr` / `table tr`, фильтрует строки с < 5 ячеек
- **`extractSellerFromRow()`** — фиксированные индексы: td[1]=имя, td[2]=бренд, td[4]=страна, с fallback сканированием
- **`parseNameCell()`** — ищет `[class*="bu7"]` (имя) и `[class*="ub7"]` (ИНН), fallback: leaf divs → INN regex
- **`findMenuButton()`** — кнопка в последнем `<td>` строки (SVG без текста)
- **`findTextareaInModal()`** — ищет `form textarea`, затем sidebar с «Жалоба на» → textarea, затем любой textarea; 5 попыток; debug-лог форм и sidebar при неудаче
- **`findComplaintSidebar()`** — новая функция: находит контейнер с заголовком «Жалоба на ...» и формой внутри
- **`findSubmitButton()`** — ищет `form button[type="submit"]` с текстом «Отправить» в sidebar, затем в любой форме
- **`closeModal()`** — ищет `button[type="button"]` с SVG (×) в sidebar, затем back-arrow, затем Escape
- **`attachFile()`** — ищет `input[type="file"]` в sidebar, затем в форме, затем на странице
- **`goToNextPage()`** — использует `ul li button` с `data-selected="true"` / `disabled` для определения текущей страницы
- **Ожидание после «Пожаловаться»** увеличено с 3с до 5с (sidebar открывается медленно)
- **Пагинация** ожидание увеличено до 3с после клика на страницу
