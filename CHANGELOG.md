# Changelog

## [3.6.1] - 2026-03-16

### Fixed
- **Собственные товары попадали в дубликаты** — при пакетном сборе в `collectCompetitors()` исключался только текущий SKU, остальные 67 собственных товаров находились как «конкуренты». Теперь ВСЕ собственные SKU (весь массив `skus[]`) добавляются в `seenSkus` перед сбором (`content/content-duplicates.js`)
- **Вайтлист фильтровал ВСЕХ конкурентов** — `applyWhitelist()` при проверке по продавцу использовал `val.includes(item.seller.toLowerCase())`, и когда `seller` пустая строка `""`, `val.includes("")` всегда `true` → все товары отфильтровывались. Добавлена проверка: пустой `seller` не матчится по вайтлисту (`content/content-duplicates.js`)

### Changed
- **Лог вайтлиста** — сообщение теперь показывает сколько пропущено и сколько осталось: «Вайтлист: пропущено X из Y, осталось Z» (было неоднозначное «Отфильтровано по вайтлисту: N товаров»)
- **Version** — 3.6.0 → 3.6.1

## [3.6.0] - 2026-03-16

### Fixed
- **Пакетный сбор собирал неполные данные из-за неверного скроллинга** — анализ реального HTML страницы «Список товаров» показал: виртуальный скроллинг привязан к `window.scrollTo`, а не к контейнеру с `overflow-y`. Старый `findScrollContainer()` находил `document.documentElement` как фоллбэк, но `scrollTarget.scrollTop` работал некорректно для window. Полная перезапись `content/content-batch-products.js` v3.5.0 → v3.6.0

### Changed
- **`scrollPageAndCollect()`** — заменяет `scrollAndCollectPageSkus()`. Двухфазный сбор: Фаза 1 — прямой DOM-запрос (если все строки уже в DOM); Фаза 2 — инкрементальная прокрутка `window.scrollTo()` с абстрактным слоем scroll-операций через closures (`getScrollTop/getScrollHeight/getViewHeight/doScroll`)
- **`findScrollableElement(table)`** — заменяет `findScrollContainer()`. Возвращает `null` когда нет контейнера с overflow → сигнал использовать window
- **`collectVisibleSkus(statusIdx, seenSkus)`** — заменяет `parseVisibleRows()`. Упрощённая дедупликация только по SKU (без текстового ключа строки)
- **`getExpectedCountFromTabs()`** — NEW: считывает ожидаемое количество товаров из badge вкладки «Все» (`.rc8110-a0`), досрочное завершение при достижении
- **`goToNextPage()`** — скроллит страницу до конца перед поиском кнопок пагинации (кнопки могут быть вне viewport)
- **`waitForTableReady()`** — заменяет `waitForTableStable()`. Учитывает и `table tbody tr` и `data-widget` SKU-spans
- **Диагностика** — расширенное логирование: тип скроллинга (window/container), scrollHeight, viewHeight, позиция, количество шагов без новых строк
- **Version** — 3.5.0 → 3.6.0

## [3.5.0] - 2026-03-16

### Fixed
- **Пакетный сбор собирал только видимые строки (17 из 36)** — таблица товаров на seller.ozon.ru использует виртуальный скроллинг: в DOM одновременно ~9-10 строк из ~36 на странице. Полная перезапись `content/content-batch-products.js`: добавлен `scrollAndCollectPageSkus()` — инкрементальная прокрутка контейнера таблицы с интервалом 300мс, сбор строк на каждой позиции скролла, дедупликация по SKU и текстовому ключу строки
- **Двойное выполнение скрипта** — усилена тройная защита от повторной инъекции: `window.__obgBatchLoaded` + DOM-guard `#__obg-batch-guard` + немедленная установка `isCollecting = true` в обработчике сообщений (до async вызова)

### Added
- **Стратегия «Проверка из таблицы»** — новый 4-й режим сбора SKU: загрузка XLSX-файла «Цены товаров» из seller.ozon.ru → Шаблоны → Цены товаров. Расширение парсит файл локально (без отправки), извлекает SKU из колонки B листа «Товары и цены», фильтрует по статусу и запускает поиск дубликатов (`popup/popup.html`, `popup/popup.js`)
- **`libs/xlsx-reader.js`** — минимальный XLSX-парсер для Chrome Extension: ZIP-распаковка через `DecompressionStream('deflate-raw')`, парсинг sharedStrings.xml и sheet XML через DOMParser. Автоматически находит лист с заголовком «SKU», пропускает неактивные товары (Не продается, Убран из продажи, Заблокирован, Архив)
- **`findScrollContainer()`** — поиск прокручиваемого контейнера таблицы вверх по DOM-дереву (`overflow-y: auto|scroll`)
- **`parseVisibleRows()`** — парсинг видимых строк таблицы с дедупликацией по SKU и текстовому ключу
- **5 стратегий поиска кнопки Next** — `goToNextPage()` больше не зависит от `findPaginationContainer()`, ищет ВСЕ кликабельные элементы ниже таблицы: нумерованная кнопка currentPage+1, aria-label next, текстовые стрелки «›»/«»», SVG-стрелка, следующая по DOM

### Changed
- **Version** — 3.4.0 → 3.5.0

## [3.4.0] - 2026-03-16

### Fixed
- **Пагинация собирала только 2 страницы** — `goToNextPage()` использовал `data-selected`/`aria-current` атрибуты, которые OZON seller panel не выставляет стабильно. Полная перезапись: мульти-стратегия определения текущей страницы (атрибуты, CSS-классы, `getComputedStyle` fontWeight/backgroundColor, disabled-состояние). Контейнер пагинации ищется ПОД таблицей через `getBoundingClientRect()`, исключая табы над ней (`content/content-batch-products.js`)

Добавлен `allFrames: false` в `chrome.scripting.executeScript` для batch/duplicates/legacy скриптов (`background/service-worker.js`). Усилена защита: `window.__obgBatchLoaded` флаг + DOM-guard (`content/content-batch-products.js`)
- **Пауза терялась при закрытии popup** — после паузы и закрытия/открытия popup кнопка «Продолжить» пропадала. Состояние паузы теперь сохраняется в `chrome.storage.local` ключ `obgPausedSession` с полным снимком сессии (SKU, индекс, результаты, конфиг). ### Added
- **Кнопка копирования SKU** — иконка 📋 рядом с каждым SKU дубликата в таблице отчёта и быстрых результатах. Клик копирует артикул в буфер, иконка на секунду меняется на ✓ (`popup/popup.js`, `popup/popup.css`)
- **Карточка возобновления** — при наличии сохранённой приостановленной сессии на вкладке «Дубликаты» появляется карточка с информацией (дата, прогресс, найденные дубликаты) и кнопками «▶ Продолжить сканирование» / «✕ Отменить» (`popup/popup.html`)
- **`resumeFromSaved` action** — новый обработчик в service-worker: восстанавливает `duplicateScanState` из `obgPausedSession` и продолжает обработку с сохранённого индекса
- **`getPausedSession` action** — новый обработчик для проверки наличия приостановленной сессии
- **`savePausedSession()`/`clearPausedSession()`/`loadPausedSession()`** — функции персистентности паузы в `background/service-worker.js`

### Changed
- **`waitForTableStable()`** — увеличено количество раундов стабилизации с 3 до 5 (2.5с вместо 1.5с) для более надёжной загрузки таблицы
- **README.md** — полная перезапись под v3.x: поиск дубликатов как основная функция, 3 стратегии, пауза/продолжение, экспорт, структура проекта
